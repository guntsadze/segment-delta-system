# 🚀 Segment Delta System

**NestJS | Next.js | Redis | BullMQ**

---

## 🎯 გამოწვევა (The Problem)

- **Scalability:** როგორ გავფილტროთ მილიონობით ტრანზაქცია რეალურ დროში?
- **Overload:** როგორ ავიცილოთ თავიდან ბაზის დატვირთვა ხშირი ცვლილებებისას?
- **Accuracy:** როგორ დავაფიქსიროთ ზუსტი სხვაობა (**Delta**) წევრობებს შორის?
- **Automation:** როგორ ვმართოთ რთული დამოკიდებულებები სეგმენტებს შორის?

---

## 🏗️ არქიტექტურული ხედვა

- **Event-Driven:** სისტემა რეაგირებს მოვლენებზე (ტრანზაქცია, დროის შეცვლა).
- **Decoupled:** API, ფონური პროცესები (Worker) და UI განცალკევებულია.
- **Asynchronous:** მძიმე გამოთვლები ხდება ფონურ რეჟიმში რიგების მეშვეობით.

---

## 🛠️ ტექნოლოგიური Stack

| შრე           | ტექნოლოგია     | როლი                                  |
| :------------ | :------------- | :------------------------------------ |
| **Backend**   | NestJS         | მოდელირებული და სკალირებადი API       |
| **Frontend**  | Next.js 14     | რეალური დროის ინტერფეისი (App Router) |
| **Database**  | PostgreSQL     | მონაცემთა ბაზა (Prisma ORM)           |
| **Queuing**   | BullMQ & Redis | დატვირთვის მართვა და დედუპლიკაცია     |
| **Real-time** | Socket.IO      | მყისიერი სინქრონიზაცია ბრაუზერთან     |

---

## 🧠 ძრავი: Evaluator & Delta

- **Evaluator:** იყენებს **Raw SQL აგრეგაციებს** (`GROUP BY`, `HAVING`).
  - _შედეგი:_ 100-ჯერ უფრო სწრაფია, ვიდრე კოდში ფილტრაცია.
- **Delta Service:** იყენებს სიმრავლეთა თეორიას (**Set Theory**).
  - პოულობს ზუსტ სხვაობას: `Added` და `Removed`.
  - უზრუნველყოფს მონაცემთა სიმყარეს ტრანზაქციების მეშვეობით.

---

## ⚡ ოპტიმიზაცია: BullMQ & Redis

- **Job Deduplication (`jobId`):**
  - 500 ტრანზაქცია წამში = **1 ოპტიმიზებული გადათვლა**.
- **Debouncing:** სისტემა აერთიანებს ცვლილებებს და არ ტვირთავს ბაზას.
- **Backpressure:** Worker-ი ამუშავებს დავალებებს იმ სისწრაფით, რა სისწრაფესაც ბაზა უძლებს.

---

## 🔗 კასკადური განახლებები (Cascading)

- **ლოგიკა:** სეგმენტები შეიძლება ეყრდნობოდეს ერთმანეთს (`IN_SEGMENT`).
- **ავტომატიზაცია:** ერთი სეგმენტის "დელტა" ავტომატურად ააქტიურებს მასზე დამოკიდებულ სეგმენტებს.
- **უსაფრთხოება:** გამოყენებულია **Snapshot-based evaluation** ციკლური დამოკიდებულებების თავიდან ასაცილებლად.

---

## 📡 რეალური დრო და მონიტორინგი

- **Socket.IO Rooms:** ტრაფიკი იგზავნება მხოლოდ დაინტერესებულ მომხმარებლებთან.
- **Enriched Logs:** ბექენდი ამზადებს ფრონტისთვის მესიჯებს ლოგებისთვის.
- **Bull Board:** ვიზუალური რეიდისის პანელი რიგების მონიტორინგისთვის.

---

## 🧪 სიმულატორი

- **Advance Time:** ბაზაში თარიღების გადაწევა ხელოვნურად.
- **Stress Test:** 50,000 იუზერის იმპორტი პორციებად (**Chunking**).
- **Manual Refresh:** სტატიკური სეგმენტების (Snapshot) მართვის შესაძლებლობა.
